# Лабораторная работа №1. Последовательное умножение квадратных матриц

## 1. Постановка задачи

Разработать программу на языке C++ для перемножения двух квадратных матриц.
Реализация должна быть **однопоточной**, выполнена **вручную циклами** (без использования
библиотечных функций умножения), с порядком обхода **i-k-j**.

**Требования:**
- Исходные данные — файлы с матрицами
- Выходные данные — файл с результирующей матрицей, время выполнения, объём задачи
- Автоматизированная верификация результатов (Python + NumPy)
- Эксперименты для размеров: 200, 400, 800, 1200, 1600, 2000

## 2. Описание алгоритма

### 2.1 Порядок обхода i-k-j

Стандартный алгоритм умножения матриц C = A × B:

```
C[i][j] = Σ(k=0..N-1) A[i][k] * B[k][j]
```

В реализации используется порядок циклов **i-k-j**, который является более
кэш-дружественным по сравнению со стандартным i-j-k:

```cpp
for (int i = 0; i < n; i++) {
    for (int k = 0; k < n; k++) {
        double a_ik = A[i * n + k]; // загружаем один раз
        for (int j = 0; j < n; j++) {
            C[i * n + j] += a_ik * B[k * n + j];
        }
    }
}
```

**Преимущества i-k-j:**
- Внутренний цикл по `j` обращается к `B[k][j]` и `C[i][j]` последовательно по строке
- Оба массива читаются/записываются с единичным шагом (stride=1)
- Значение `A[i][k]` вынесено во временную переменную — исключается повторное обращение к памяти

### 2.2 Хранение матриц

Матрицы хранятся в одномерных массивах (`new double[n*n]`) с линейной индексацией `[i*n+j]`.

### 2.3 Вычислительная сложность

- Количество операций: **2N³** (N³ умножений + N³ сложений)
- Сложность по памяти: **3N²** элементов типа `double` (24N² байт)

## 4. Верификация

Результат умножения проверяется с помощью Python-скрипта `verify.py`, который:
1. Загружает матрицы A, B и результат C++ из файлов
2. Вычисляет эталонное произведение `C_ref = A @ B` через NumPy (использует оптимизированный BLAS)
3. Сравнивает результаты по:
   - Максимальной абсолютной ошибке
   - Средней абсолютной ошибке
   - Относительной ошибке по норме Фробениуса

Критерий прохождения: относительная ошибка < 10⁻⁶.

## 6. Анализ результатов

### 6.1 Временная сложность

Экспериментальные данные подтверждают теоретическую сложность **O(N³)**:
- При увеличении N в 2 раза (например, с 400 до 800) время увеличивается примерно в 8 раз
- Кривая фактического времени хорошо ложится на теоретическую кривую N³

### 6.2 Производительность

Производительность в GFLOP/s остаётся относительно стабильной для всех размеров матриц
(~3.0–3.4 GFLOP/s), что характерно для однопоточного кода с хорошей кэш-локальностью.

Порядок обхода i-k-j обеспечивает последовательный доступ к памяти во внутреннем цикле,
что минимизирует промахи кэша.

### 6.3 Память

Потребление памяти растёт как **O(N²)** — хранятся три матрицы размером N×N:
- Для N=200: ~0.9 МБ
- Для N=2000: ~91.6 МБ

## 7. Выводы

1. **Реализован** алгоритм умножения квадратных матриц с порядком обхода i-k-j,
   обеспечивающий кэш-дружественный последовательный доступ к элементам матриц.

2. **Подтверждена корректность** — все результаты прошли автоматическую верификацию
   через NumPy с относительной ошибкой менее 10⁻⁶.

3. **Экспериментально подтверждена** теоретическая сложность O(N³) — время
   выполнения растёт кубически с увеличением размера матрицы.

4. **Производительность** однопоточной реализации составляет ~3.0–3.4 GFLOP/s,
   что существенно ниже пиковой производительности современных процессоров.
   Это создаёт мотивацию для распараллеливания в последующих лабораторных работах.

5. **Потребление памяти** линейно зависит от N² и составляет от ~1 МБ для
   малых матриц до ~92 МБ для N=2000, что не является ограничивающим фактором.
